# 文风规范

## 基础风格

| 项目 | 要求 |
| --- | --- |
| 叙事视角 | 默认第三人称有限视角 |
| 语言质感 | 清晰、具体、有画面感 |
| 节奏 | 场景推进优先，少用空泛概括 |
| 情绪 | 让读者通过行动和选择感受情绪 |
| 信息密度 | 设定信息分散进入场景 |

## 场景写法

- 每个重要场景要有明确地点、人物、目标、阻力和结果。
- 少写抽象判断，多写可观察动作。
- 环境描写服务于情绪、危险、信息或人物选择。
- 打斗、追逐、谈判、试炼等场景要有阶段变化。
- 关键转折前要有足够铺垫，转折后要有即时反应。

## 对白规范

- 对白必须推动关系、冲突、信息差或人物性格。
- 不用长篇对白解释整套设定。
- 不让多个角色说同一种语气。
- 重要人物应有稳定的表达习惯。
- 对白后要适度加入动作、停顿、表情或场景反馈。

## 爽点规范

- 爽点来自压迫、选择、代价、反击、揭示和兑现。
- 主角可以赢，但不能无准备、无代价、无逻辑地赢。
- 每次胜利要带来新的麻烦、注意、敌意或责任。
- 打脸场景要先建立误判、羞辱或规则压迫。

## 描写比例

| 内容 | 建议比例 |
| --- | --- |
| 行动与场景 | 40% 到 55% |
| 对白 | 25% 到 40% |
| 心理 | 10% 到 20% |
| 设定说明 | 不超过 10%，并尽量嵌入场景 |

## 禁止文风

- 机械总结剧情。
- 连续使用空泛形容词。
- 用旁白替角色做判断。
- 重复解释已经明确的信息。
- 频繁使用“他知道”“他明白”“所有人都震惊了”这类弱表达。
