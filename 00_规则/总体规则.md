# 总体规则

## 设定优先级

当文件之间出现冲突时，按以下顺序处理：

1. `00_规则/` 中的硬规则。
2. `01_小说设定/小说定位.md`。
3. `03_剧情规划/全书大纲.md` 和 `03_剧情规划/分卷大纲.md`。
4. `04_动态记忆/` 中已发生事实。
5. `03_剧情规划/当前章节卡.md`。
6. 临时创意和单章灵感。

已发生正文事实优先于未执行的大纲。大纲可以调整，但必须记录调整原因。

## 连续性原则

- 人物不能忘记自己已经知道的信息。
- 人物位置、伤势、境界、资源和关系必须连续。
- 势力行动要符合资源、距离、情报和利益。
- 修炼突破必须有积累、契机、代价或风险。
- 伏笔必须有表层意义，不能只为未来强行服务。

## 单章最低要求

每章至少满足以下两项：

- 推进主线或分卷目标。
- 改变人物关系或人物处境。
- 揭示新信息或制造新问题。
- 完成一次冲突、选择、交易、战斗、追查或反转。
- 埋设或回收一个伏笔。

## 新增设定规则

新增设定必须同时满足：

- 不破坏已有世界观。
- 不让主角无代价变强。
- 不让重要冲突失去压力。
- 能记录到设定文件、人物文件或动态记忆。

## 修改规则

修改正文时必须记录：

- 修改章节。
- 修改原因。
- 涉及设定。
- 影响后续剧情的地方。

## 章节完成标准

- 本章目标已完成或被明确阻断。
- 出场人物行为符合当前状态。
- 没有明显设定冲突。
- 结尾有继续阅读的理由。
- 动态记忆已更新。
